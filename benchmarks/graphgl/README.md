# Graph.gl / Deck.gl Proof of Concept

WebGL-based phylogenetic tree visualization using Graph.gl (built on Deck.gl).

## Features

- **Performance Testing**: Load time, render time, and graph metrics displayed in UI
- **Phylogenetic Trees**: Pre-loaded datasets from your converted Newick edgelists
- **Sample Graphs**: Random graphs and mathematical graph generators for comparison
- **Interactive Visualization**: WebGL-accelerated rendering with D3 force layout

## Available Datasets

### Phylogenetic Trees

- **Simulated 1K**: ~1,000 nodes (ete3 generated)
- **Simulated 3K**: ~3,000 nodes (ete3 generated)
- **Simulated 10K**: ~10,000 nodes (ete3 generated)
- **H. influenzae MLST**: ~3K nodes (PubMLST)
- **C. coli MLST**: ~3K nodes (PubMLST)

### Sample Graphs

- Les Misérables character network
- Random graphs (20-5000 nodes)
- Mathematical graphs (ladder, binary tree, grid, Watts-Strogatz)

## Running the Demo

```bash
npm install
npm run dev
```

Open http://localhost:5173 and select datasets from the dropdown.

## Performance Metrics

The UI displays:

- **Node Count**: Number of vertices in the graph
- **Edge Count**: Number of edges in the graph
- **Load Time**: Time to fetch and parse data (ms)
- **Render Time**: Time to initialize Graph.gl (ms)

## Data Format

Phylogenetic trees are loaded from CSV edgelists:

```csv
Source,Target
Node1,Node2
Node2,Node3
```

Generated by `analysis/newick_to_edgelist.py` from Newick tree files.
